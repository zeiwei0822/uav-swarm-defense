# 無人機機群攻防模擬 — 台灣海峽防衛戰

以 Python 打造的無人機**機群攻防**模擬：對岸機群跨海峽進襲、我方防空攔截，並用 **AI** 識別指揮節點、預測軌跡。專題核心在比較「**傳統防空 vs AI 防空**」，凸顯 AI 的價值與限制。

## 🔗 線上展示（手機可看）

**https://zeiwei0822.github.io/uav-swarm-defense/**

一頁式對比：同一場景（環形陣）下「**傳統防空放 13 架突防**」vs「**AI 防空 0 架突防**」，可切換觀看兩場戰役。

## 專題四要點

1. **機群陣型與飛行路徑分析** — V字／楔形／縱隊／方陣／環形，各有隱蔽性與通訊韌性的取捨。
2. **領機／中繼機失效處理** — 心跳逾時 → 選舉 → 中繼機遞補 → 編隊重組（失效韌性）。
3. **AI 軌跡預測** — LSTM 預測機群未來位置供攔截導引（勝卡爾曼濾波約 49%，ADE ≈ 6.6 m）。
4. **AI 找出領機／中繼機** — RandomForest 從 9 維飛行行為特徵識別指揮節點（領機 Top-1 ≈ 75%）。

## 戰術與對抗

- **多軸夾擊**：機群分三股縱隊、向心夾擊推進。
- **誘餌／沉默領機**（攻方反制 AI 識別）：誘餌衝前方扮假領機、真領機藏身 → AI 多半把火力打在誘餌上 → 示範「行為識別」的死穴。
- **指揮節點打擊（斬首）**：AI 認出領機後優先擊殺、打斷指揮鏈，達成傳統防空做不到的「癱瘓整群」。

## 執行（桌面控制台）

需求：Python 3.10、`numpy / matplotlib / scikit-learn / torch / Pillow`。

```bash
# Windows 請先設定 UTF-8 避免 cp950 編碼錯誤
set PYTHONUTF8=1            # PowerShell: $env:PYTHONUTF8='1'

python train.py            # 首次：產生 models/（訓練 LSTM / RandomForest / MLP，約數分鐘）
python app.py              # 開啟整合控制台（選陣形/防方/誘餌 → 開戰）
```

控制台操作：上排選**陣形**、**防方（傳統／AI）**、**誘餌**數、種子 → 按「開戰」。
跑完用底部播放列觀戰（上一/下一重點、播放、0.25～4× 變速、導覽模式自動暫停解說）。
其他：`python run_sim.py`（單場／匯出動畫）、`python analyze.py`（分析圖）。

## 架構

| 模組 | 說明 |
|---|---|
| `core/swarm.py` | 機群階層通訊、失效偵測/選舉/遞補/重組、多軸夾擊、誘餌/沉默領機 |
| `core/defense.py` | 雷達、AI 識別排程、火控策略（傳統最近 / AI 指揮節點打擊）、攔截彈導引 |
| `core/formations.py` | 5 種陣型庫與隱蔽/通訊取捨 |
| `core/engine.py` | 攻防主循環、全程記錄器、戰報 |
| `ai/` | LSTM 軌跡預測（`trajectory.py`）、RF/MLP 角色識別（`identify.py`）、訓練資料生成（`datagen.py`） |
| `viz/tactical2d.py` | 2D 俯視戰術地圖＋即時遙測側欄＋資料驅動分幕旁白 |
| `app.py` | tkinter 整合控制台（固定海峽視角） |
| `docs/` | 線上展示網頁（GitHub Pages） |

## 已驗證成果

- LSTM 軌跡預測 ADE 6.6 m，勝卡爾曼 CV 約 49%（2.5 s 預測時域）。
- 領機 Top-1 識別：RandomForest ≈ 75%、MLP ≈ 71%、規則法 ≈ 27%（跨陣型、含多軸）。
- 環形陣 seed 17：傳統防空 13 架突防 vs AI 防空 0 架突防（AI 認出中央領機 → 斬首 → 機群癱瘓）。
- 誘餌（沉默領機）開啟後：AI 約 6 成火力被導到誘餌、真領機存活率大增、突防上升。
